What is execution context

the java script Engines creates the Special environment which handles the transformation
and execution of js code this environment is called execution context

1)global execution context
2)Functional execution context ----> GEC:- when ever a js engines encounter a js file it creates default execcution contrext
called global execution context which executes all the code except the code inside the function ----> FEC :- whenever a function is called js engine creates different FEC in GEC to executes code inside the functions

it determines the values of the variable and functions and setups the scope chain for context execution
initially it stores all the variable as undefined and function as defined
execution phase :- in this phase the js code start executing the code line by line it assign the
value to all the variable which is defined as undefined and the skips the funstion until its called then when the function is called it creats FEC where the code inside the function is executed after execution of the first funtion the function is removed from call stack and the second function executed after all the functions are executed the fec is removed from call stack and at the end GEC is also removed from call stack after execution

what is temporaldead zone(TDZ)

A let or const variable is said to be in a "temporal dead zone" (TDZ) from the start of the block until code
execution reaches the line where the variable is declared and initialized. While inside the TDZ, the
variable has not been initialized with a value, and any attempt to access it will result in a ReferenceError 

              if(true){
                console.log(a); //a is in temporal dead zone

                let a=12;
              }
